# WebSocket Relay — удаленный доступ к устройствам за NAT

Система для организации удаленного доступа к веб-интерфейсам устройств, находящихся за NAT, через центральный relay-сервер с белым IP.

## 🚀 Как это работает

Устройства за NAT не имеют публичного IP-адреса и недоступны напрямую из интернета. WebSocket Relay решает эту проблему:

1. **Агент** на устройстве самостоятельно устанавливает **исходящее WebSocket-соединение** к relay-серверу с белым IP
2. **Relay-сервер** держит это соединение открытым и использует его как канал для передачи данных в обе стороны
3. **Клиенты** (браузеры, curl) отправляют HTTP-запросы на relay-сервер
4. Relay-сервер преобразует HTTP-запрос в WebSocket-сообщение и передает его агенту
5. Агент обрабатывает запрос (обращаясь к локальному веб-серверу устройства) и возвращает ответ через тот же WebSocket-канал

Таким образом, устройство за NAT становится доступным из интернета без настройки проброса портов на роутере.

## 🏗️ Архитектура

```
┌─────────────────┐                    ┌─────────────────────────────────────┐
│   Интернет      │                    │      Сервер с белым IP              │
│                 │                    │     хх.ххх.ххх.хх                   │
│  ┌───────────┐  │                    │                                     │
│  │  Клиент   │  │   HTTP запрос      │  ┌─────────────────────────────┐    │
│  │  (curl,   │──┼───────────────────▶│  │      Relay-сервер           │    │
│  │   браузер)│  │  :8080             │  │      (relay.py)             │    │
│  └───────────┘  │                    │  │                             │    │
│                 │                    │  │  • HTTP API  :8080          │    │
│                 │                    │  │  • WebSocket :8765          │    │
│                 │                    │  └─────────────┬───────────────┘    │
│                 │                    │                │                    │
│                 │                    │                │ WebSocket          │
│                 │                    │                │ (постоянное        │
│                 │                    │                │  соединение)       │
│                 │                    │                ▼                    │
│                 │                    │     ┌──────────────────────┐        │
│                 │                    │     │   Устройство за NAT  │        │
│                 │                    │     │   (агент + веб-      │        │
│                 │                    │     │    интерфейс)        │        │
│                 │                    │     │                      │        │
│                 │                    │     │  • agent.py          │        │
│                 │                    │     │  • веб-сервер :80    │        │
│                 │                    │     └──────────────────────┘        │
└─────────────────┘                    └─────────────────────────────────────┘
```

**Поток данных:**
1. Агент подключается к relay-серверу по WebSocket (ws://xx.xx.xx.xx:8765)
2. Клиент отправляет HTTP-запрос на relay-сервер (http://xx.xx.xx.xx:8080/test)
3. Relay-сервер передает запрос агенту через WebSocket
4. Агент обрабатывает запрос (локально) и возвращает ответ
5. Relay-сервер отдает ответ клиенту

## 📦 Требования

### Relay-сервер:
- Python 3.7+
- Белый (публичный) IP-адрес
- Открытые порты: 8080 (HTTP), 8765 (WebSocket)

### Агент (устройство):
- Python 3.7+
- Доступ в интернет (для подключения к relay-серверу)
- Локальный веб-сервер (опционально, для тестов можно использовать встроенный)

## 🔧 Установка и настройка

### 1. Relay-сервер (на сервере с белым IP)

#### 1.1. Подготовка окружения

```bash
# Создаем директорию для проекта
mkdir ~/relay-server
cd ~/relay-server

# Создаем виртуальное окружение
python3 -m venv venv

# Активируем виртуальное окружение
source venv/bin/activate

# Обновляем pip
pip install --upgrade pip
```

#### 1.2. Установка зависимостей

```bash
pip install aiohttp websockets
```

#### 1.3. Открытие портов в фаерволе

```bash
# Если используем ufw (Ubuntu/Debian)
sudo ufw allow 8080/tcp
sudo ufw allow 8765/tcp
sudo ufw reload

# Если используем iptables
sudo iptables -A INPUT -p tcp --dport 8080 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 8765 -j ACCEPT
```

**Важно:** Если сервер в облаке (AWS, GCP, DigitalOcean), нужно также открыть порты 8080 и 8765.

```python
RELAY_URL = "ws://xx.xx.xx.xx:8765"  # Замените на ваш IP
```

## 🚀 Запуск

### 1. Запуск relay-сервера

```bash
cd ~/relay-server
source venv/bin/activate
python relay.py
```

Ожидаемый вывод:
```
HTTP relay: http://0.0.0.0:8080
WS relay: ws://0.0.0.0:8765
```

### 2. Запуск агента на устройстве

```bash
cd ~/agent
source venv/bin/activate
python agent.py
```

Ожидаемый вывод:
```
==================================================
📱 Агент для устройства: blazer-1
🔗 Relay-сервер: ws://xx.xx.xx.xx:8765
==================================================
🔌 Подключение к ws://xx.xx.xx.xx:8765...
✅ WebSocket подключен!
📤 Отправлена регистрация: blazer-1
```

На relay-сервере должно появиться:
```
Agent connected
```

## 🧪 Тестирование

### 1. Проверка через curl

С любого компьютера (или с самого сервера) выполните запрос к reley-серверу:

```bash
curl http://xx.xx.xx.xx:8080/test
```

Ожидаемый ответ:
```json
{
    "status": "ok",
    "device": "blazer-1",
    "method": "GET",
    "url": "/test",
    "message": "Hello from device behind NAT!",
    "timestamp": 1742500000.123
}
```

### 2. Проверка через браузер

Откройте браузер и перейдите по адресу:
```
http://xx.xx.xx.x:8080/
```

Вы должны увидеть JSON-ответ от агента.

### 3. Просмотр логов

**На relay-сервере:**
```
Agent connected
```

**На агенте:**
```
📨 Получено сообщение от relay: {"requestId": "xxx", "method": "GET", ...
📥 HTTP запрос: GET /test
📤 Ответ отправлен для xxx
```

## 🌐 Настройка локального веб-сервера

По умолчанию агент возвращает тестовый JSON-ответ. Чтобы проксировать реальный веб-интерфейс устройства:

### Вариант 1: Использовать существующий веб-сервер

Раскомментируйте в agent.py блок с `requests.request` и закомментируйте тестовый ответ:

```python
# ВАРИАНТ 1: Если есть локальный веб-сервер
resp = requests.request(
    method=req["method"],
    url=f"http://localhost:80{req['url']}",  # или https://localhost:443
    headers=req["headers"],
    data=body,
    timeout=10
)

response = {
    "requestId": req["requestId"],
    "status": resp.status_code,
    "headers": dict(resp.headers),
    "body": base64.b64encode(resp.content).decode()
}
```

### Вариант 2: Запустить тестовый веб-сервер

На устройстве (в отдельном терминале):

```bash
# HTTP-сервер на порту 80 (требует sudo)
sudo python3 -m http.server 80

# Или на порту 8080 (без sudo)
python3 -m http.server 8080
```

Соответственно измените порт в агенте на 8080.

## 🔒 Безопасность

### 1. Ограничение доступа

Для production-окружения рекомендуется:

- Использовать HTTPS вместо HTTP (добавить Caddy или nginx перед relay)
- Добавить аутентификацию на relay-сервере
- Использовать WSS (WebSocket Secure) вместо WS

### 2. Проверка сертификатов

Если устройство использует HTTPS с самоподписанным сертификатом, в агенте используется `verify=False`. Для production используйте корректные сертификаты.

## 🛠️ Устранение неполадок

### Проблема: curl возвращает "No agent connected"

**Причина:** Агент не подключен к relay-серверу.

**Решение:**
1. Проверьте, запущен ли агент
2. Проверьте IP в `RELAY_URL` в agent.py
3. Проверьте, открыт ли порт 8765 на сервере:
   ```bash
   sudo netstat -tlnp | grep 8765
   sudo ufw status
   ```

### Проблема: WebSocket connection failed

**Причина:** Сетевая недоступность или блокировка порта.

**Решение:**
```bash
# Проверка доступности порта с устройства
telnet xx.xx.xx.xx 8765
nc -zv xx.xx.xx.xx 8765

# Проверка фаервола на сервере
sudo ufw allow 8765/tcp
sudo ufw reload
```

### Проблема: Agent не получает запросы

**Причина:** Агент не отправил регистрацию или relay не видит подключение.

**Решение:**
1. Проверьте, что в agent.py есть отправка регистрации:
   ```python
   register_msg = {"type": "register", "deviceId": DEVICE_ID, ...}
   await ws.send(json.dumps(register_msg))
   ```
2. Проверьте логи relay-сервера — должно быть "Agent connected"

### Проблема: Ошибка кодировки в agent.py

**Решение:**
```bash
# Проверьте кодировку файла
file agent.py

# Если не UTF-8, перекодируйте
iconv -f ISO-8859-1 -t UTF-8 agent.py > agent_clean.py
mv agent_clean.py agent.py
```

### Проблема: ModuleNotFoundError

**Решение:** Убедитесь, что зависимости установлены в виртуальном окружении:
```bash
source venv/bin/activate
pip list | grep -E "(aiohttp|websockets|requests)"
```

Если пакетов нет:
```bash
pip install aiohttp websockets  # для relay
pip install websockets requests  # для агента
```
